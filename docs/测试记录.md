## 5. 测试中发现并修复的问题

### 5.1 测试用例缺少机器可检索验证层级字段

现象：

`05_test_cases.md` 中有中文标题“架构设计验证用例”，但没有显式写入 `architecture_verification` 字段，导致后续记忆检索和自动检查不稳定。

解决：

- 在每条测试用例中增加 `verify_level` 字段。
- 测试用例表头改为包含 `case_id / verify_level / target / case_type / steps / expected`。

### 5.2 失败任务被写入成功路径记忆

现象：

人工恢复流程中，如果没有开启代码执行，编码状态为 `plan_only`，最终状态应为失败或未完成，但旧逻辑只要生成了闭环摘要就写入 `success_paths`。

解决：

- 修改 `run_workflow`，只有 `final_key_path.status == success` 时才写入 `success_paths`。
- 增加回归测试：未执行编码时不允许进入成功路径记忆。

### 5.3 大模型调用没有贯穿全流程

现象：

最初完整端到端测试使用的是 `template` 模式，只能证明 LangGraph 流程、文件产物、编码、测试和记忆管理能跑通；百炼只通过 `llm-smoke` 做了单次冒烟测试，不能说明“大模型参与完成了全流程”。

解决：

- 在 `bailian` 模式下，让以下阶段都真实调用百炼模型：
  - `requirement_review`
  - `architecture_review`
  - `detailed_design`
  - `test_design`
  - `test_execution`
  - `closed`
- 每次模型调用写入 `state_latest.json` 的 `llm_calls`。
- 每个阶段 Markdown 文件写入“大模型阶段判断”或“大模型闭环总结”。
- 补充 `llm_calls` 到 LangGraph `AgentState`，避免节点间状态丢失。
- 为大模型调用增加 3 次重试，解决偶发 `Connection error`。
